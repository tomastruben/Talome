## The Stack Explained

A self-hosted media server is a pipeline. Each app handles one stage:

<Mermaid chart={`graph LR
    P[Prowlarr] -->|Indexers| S[Sonarr]
    P -->|Indexers| R[Radarr]
    S -->|Download request| Q[qBittorrent]
    R -->|Download request| Q
    Q -->|Completed files| M[/Media Library/]
    M --> J[Jellyfin / Plex]
    O[Overseerr] -->|TV requests| S
    O -->|Movie requests| R
`} />

| App | Role | What It Does |
|-----|------|-------------|
| **Jellyfin** or **Plex** | Media server | Streams your library to any device. Jellyfin is fully open-source; Plex has a free tier with optional premium features. |
| **Sonarr** | TV management | Monitors TV shows, searches for episodes, manages quality profiles, renames files |
| **Radarr** | Movie management | Same as Sonarr, but for movies. Separate app because movies and TV have different release patterns. |
| **Prowlarr** | Indexer manager | Centralizes your search sources (indexers). Configure indexers once in Prowlarr, sync to all *arr apps. |
| **qBittorrent** | Download client | Handles the actual file transfers. Sonarr/Radarr tell it what to download; it reports back when done. |
| **Overseerr** | Request system | A clean web UI for family and friends to request movies and TV shows. Requests flow to Sonarr/Radarr automatically. |

## Volume Configuration

Media apps need to see the same files. The key is mapping your host media directory into every container that needs it.

### Common Directory Structures

**Linux (typical NAS mount):**
```
/mnt/media/
├── Movies/
├── TV/
├── Music/
├── Downloads/
│   ├── complete/
│   └── incomplete/
└── Books/
```

**macOS:**
```
/Volumes/Media Vault/Media/
├── Movies/
├── TV Shows/
├── Downloads/
└── Music/
```

### How Volumes Map

When the AI installs an app with `volumeMounts`, it maps your host directory into the container:

| App | Host Path | Container Path | Why |
|-----|-----------|---------------|-----|
| Jellyfin | `/mnt/media` | `/media` | Scans and streams the full library |
| Sonarr | `/mnt/media` | `/media` | Moves completed TV episodes to `/media/TV` |
| Radarr | `/mnt/media` | `/media` | Moves completed movies to `/media/Movies` |
| qBittorrent | `/mnt/media/Downloads` | `/downloads` | Stores downloads here; *arr apps hardlink or move from here |

The critical rule: **Sonarr/Radarr and qBittorrent must see the same filesystem** for hardlinks to work. If Sonarr's `/media` and qBittorrent's `/downloads` are on the same host mount, file moves are instant (hardlink) instead of slow copies.

## Step-by-Step (Manual Control)

If you prefer installing and configuring each app individually, the AI supports that too.

### 1. Install the Media Server

<ChatSimulation>
  <UserMessage>Install Jellyfin and point it at /mnt/media</UserMessage>
</ChatSimulation>

The AI installs Jellyfin, maps your media volume, creates libraries for Movies and TV, and triggers an initial scan.

### 2. Install Sonarr and Radarr

<ChatSimulation>
  <UserMessage>Install Sonarr and Radarr with media at /mnt/media</UserMessage>
</ChatSimulation>

Both apps install with the correct volume mapping. The AI creates root folders pointing to `/mnt/media/TV` and `/mnt/media/Movies`.

### 3. Install Prowlarr

<ChatSimulation>
  <UserMessage>Install Prowlarr and sync indexers to Sonarr and Radarr</UserMessage>
</ChatSimulation>

Prowlarr installs. After you add indexers in Prowlarr's web UI, the AI syncs them to your *arr apps using `arr_sync_indexers_from_prowlarr`.

### 4. Install qBittorrent

<ChatSimulation>
  <UserMessage>Install qBittorrent and set it as the download client for Sonarr and Radarr</UserMessage>
</ChatSimulation>

qBittorrent installs with downloads mapped to your media drive. The AI calls `arr_add_download_client` for both Sonarr and Radarr, pointing them at qBittorrent's API.

### 5. Wire Everything

If you installed apps separately and need to connect them:

<ChatSimulation>
  <UserMessage>Wire all my media apps together</UserMessage>
</ChatSimulation>

The `wire_apps` tool auto-discovers installed media apps and configures their connections — download clients, root folders, and indexer sync.
