Audiobookshelf is a self-hosted audiobook and podcast server. It provides a beautiful listening experience across web, iOS, and Android, with features like chapter navigation, bookmarks, sleep timers, and multi-user progress tracking. Talome's integration gives the AI 13 tools to manage your libraries, search your collection, track listening progress, scan for new content, and search for and download new audiobooks.

This is one of Talome's more feature-rich integrations. The 13 tools cover the full lifecycle: creating libraries, scanning for content, searching and browsing, tracking where you left off, updating progress, searching for new audiobook releases via Prowlarr, and downloading them via qBittorrent. The AI becomes a knowledgeable librarian that knows your entire audiobook collection and your listening habits.

## Setup

### 1. Install Audiobookshelf

```
You: Install Audiobookshelf
```

The AI installs Audiobookshelf from the app store, mapping your audiobook directory if known. Audiobookshelf runs on port 13378 by default.

### 2. Initial configuration

After installation, create an admin account through the Audiobookshelf web UI. Then create an API token:

1. Sign in to Audiobookshelf
2. Go to Settings (gear icon)
3. Under your user profile, find or generate an API token

Or ask the AI to guide you:

```
You: How do I get my Audiobookshelf API token?
```

### 3. Configure settings

**Settings keys:**

| Key | Example | Description |
|-----|---------|-------------|
| `audiobookshelf_url` | `http://localhost:13378` | Audiobookshelf server URL |
| `audiobookshelf_api_key` | `eyJhbGciOi...` | API bearer token |

Add these in **Settings > Integrations** or through conversation:

## Auto-Configuration

When `wire_apps` detects a running Audiobookshelf container:

- Identifies the container by image name and port
- Saves the URL to Talome settings
- Notes that a token must be created manually (Audiobookshelf tokens are tied to user accounts)
- Prompts you to provide the API token if one isn't configured

Like Home Assistant, Audiobookshelf tokens are user-specific and cannot be auto-extracted from config files. The AI will remind you to set one up.

## Available Tools

| Tool | Description |
|------|-------------|
| `audiobookshelf_get_status` | Server status: version, uptime, number of libraries, total items, total duration, and storage usage. Quick health check for the entire server. |
| `audiobookshelf_list_libraries` | List all libraries with their name, type (book or podcast), folder paths, item count, and total duration. Shows where your content is organized. |
| `audiobookshelf_add_library` | Create a new library by specifying a name, type (book or podcast), and filesystem path. The path must be accessible inside the Audiobookshelf container. |
| `audiobookshelf_get_library_items` | List items in a specific library with title, author, duration, progress, and whether the item has been started or finished. Supports pagination and sorting. |
| `audiobookshelf_search` | Search across all libraries by title, author, narrator, or series name. Returns matching items with their metadata and your current progress on each. |
| `audiobookshelf_get_item` | Get detailed information about a specific audiobook or podcast: full metadata, chapters, files, duration, narrators, series membership, and user progress. |
| `audiobookshelf_get_progress` | Get your listening progress for a specific item: current position, percentage complete, time remaining, last listened date, and whether it's finished. |
| `audiobookshelf_update_progress` | Update your listening progress for an item. Set a specific position (in seconds), mark as finished, or mark as not started. Useful for syncing with external listening. |
| `audiobookshelf_scan_library` | Trigger a library scan to detect new files, update metadata, and refresh cover art. Scans run in the background and process new additions incrementally. |
| `audiobook_search_releases` | Search for audiobook torrent releases across all Prowlarr indexers (category Audio/Audiobook). Returns downloadable releases sorted by seeders with detected language. |
| `audiobook_download` | Download an audiobook release via qBittorrent. Takes a download URL from `audiobook_search_releases` results and sends it to qBittorrent with the "audiobooks" category. |
| `audiobook_list_downloads` | List active and completed audiobook downloads from qBittorrent (category "audiobooks"). Shows progress, speed, and ETA for active downloads. |
| `audiobook_request` | Request an audiobook -- searches Prowlarr for the best available release and auto-downloads it via qBittorrent. Picks the release with the most seeders matching the language filter. |

## Troubleshooting

### "Audiobookshelf not configured" error

**Cause:** The `audiobookshelf_url` or `audiobookshelf_api_key` setting is missing.

**Fix:** Add both in Settings > Integrations. The token is found in Audiobookshelf's user settings. If you lost it, sign into the web UI and generate a new one.

### Library scan doesn't find new files

**Cause:** The files are on the host but the path isn't mounted into the Audiobookshelf container.

**Fix:** Check the container's volume mounts: `"What paths does Audiobookshelf have access to?"`. If the path is missing, add a volume mount: `"Add /media/audiobooks as a volume mount to Audiobookshelf"`.

### Metadata is wrong or missing

**Cause:** File naming doesn't match expected patterns, or online metadata lookup failed.

**Fix:** Audiobookshelf expects folders named `Author/Book Title/` with audio files inside. If your files are named differently, consider renaming. You can also manually match books through the Audiobookshelf web UI.

### Progress not syncing between devices

**Cause:** The mobile app may be offline or the API token has expired.

**Fix:** This is typically a client-side issue. Verify the server is reachable: `"Is Audiobookshelf running?"`. If the server is fine, try signing out and back in on the mobile app to refresh the token.

### Search returns no results for a book you have

**Cause:** The book's metadata might not match your search query, or it might be in a different library than expected.

**Fix:** Try searching by author name instead of title, or use a partial title. The AI searches across all libraries, so if the book exists anywhere in Audiobookshelf, it will be found.
